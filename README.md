# synapse-os

An engineering portfolio for **Kishor Sanjay Gangarde** — Software Engineer,
2+ years, backend-focused full stack developer. Built as a system that
boots, reveals architecture, and proves engineering depth rather than a
conventional "portfolio website."

```
synapse-os/
├── frontend/   React 18 + Vite + Tailwind + Framer Motion + GSAP + Zustand
├── backend/    Laravel 11 + PostgreSQL 16 + Sanctum
└── README.md
```

---

## What's implemented in this pass

**Frontend** — all 7 sections, fully built and wired to real content:
1. Boot sequence (terminal typing → name reveal → mouse-lit dot grid)
2. About process flow (scroll-linked node diagram)
3. Skills IDE panel (VS Code–style folder explorer, no bars/percentages)
4. Projects (hover blueprint → fullscreen case study: architecture diagram,
   DB schema, API flow, challenges, trade-offs, lessons, outcomes)
5. Experience git-log (commits that expand into responsibilities/impact)
6. Resume viewer (GitHub-style file card → document viewer → PDF download)
7. Contact terminal (command-driven, plus a real contact form hitting the API)

The frontend **runs standalone** even without the backend: `usePortfolioStore`
tries the Laravel API first and silently falls back to
`src/data/fallbackData.js`, which mirrors the database seeder exactly. This
means you can `npm run dev` immediately and see the full site before Postgres
or Laravel are even set up.

**Backend** — public read API is complete and functional:
- Migrations for all 6 tables (`projects`, `experiences`, `skills`,
  `about_steps`, `contact_messages`, `settings`), using `jsonb` columns for
  the structured case-study data
- Eloquent models with casts/scopes
- Controllers + routes for every public endpoint
- `PortfolioSeeder` populated with Kishor's **real** experience and projects
  (Biz Secure Labs / Net Protector, YARA Rule Portal, Malware Sample Hub, Pet
  Management System) pulled directly from his resume
- Sanctum-protected `/api/admin/*` auth boundary (login/logout + contact
  message moderation) scaffolded for a future admin panel — the admin UI
  itself was explicitly out of scope for this milestone

**Not included in this pass:** the admin panel UI, GSAP ScrollTrigger
micro-polish beyond Framer Motion's `whileInView` (which covers the same
scroll-reveal behavior with less setup), and a production Dockerfile — all
consistent with the "no admin panel / no Docker initially" constraints in
the brief.

## Backend setup (Laravel 11 + PostgreSQL 16)

This repo ships the **application code** for the backend (migrations,
models, controllers, routes, seeders, config). Laravel's own framework
skeleton (vendor/, public/index.php, storage/, remaining default config
files) is generated by Composer and is intentionally not hand-written here.

```bash
# 1. Scaffold a fresh Laravel 11 app, then copy this app code into it
composer create-project laravel/laravel synapse-os-backend "^11.0"
cd synapse-os-backend

# 2. Copy this project's backend/ files over the fresh skeleton
#    (app/, database/, routes/, bootstrap/app.php, config/cors.php, .env.example)
cp -r ../synapse-os/backend/app/* app/
cp -r ../synapse-os/backend/database/* database/
cp ../synapse-os/backend/routes/*.php routes/
cp ../synapse-os/backend/bootstrap/app.php bootstrap/app.php
cp ../synapse-os/backend/config/cors.php config/cors.php
cp ../synapse-os/backend/.env.example .env.example

# 3. Install Sanctum
composer require laravel/sanctum
php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"

# 4. Environment
cp .env.example .env
php artisan key:generate
# Edit .env: set DB_DATABASE, DB_USERNAME, DB_PASSWORD for your local Postgres 16

# 5. Create the database (example using psql)
createdb synapse_os

# 6. Migrate + seed with Kishor's real data
php artisan migrate
php artisan db:seed --class="Database\Seeders\PortfolioSeeder"

# 7. Run
php artisan serve   # http://localhost:8000
```

### API surface

| Method | Endpoint                | Description                          |
|--------|--------------------------|---------------------------------------|
| GET    | `/api/projects`          | Lightweight project list              |
| GET    | `/api/projects/{slug}`   | Full case study payload               |
| GET    | `/api/experiences`       | Experience list                       |
| GET    | `/api/skills`            | Skills grouped by category            |
| GET    | `/api/about`             | Process-flow steps                    |
| GET    | `/api/settings`          | Identity, contact, status settings    |
| POST   | `/api/contact`           | Submit a contact message (rate-limited) |
| POST   | `/api/admin/login`       | Sanctum session login                 |
| POST   | `/api/admin/logout`      | Requires auth                         |
| GET    | `/api/admin/messages`    | Requires auth — contact inbox         |

## Frontend setup (React + Vite)

```bash
cd frontend
npm install
npm run dev   # http://localhost:5173
```

The Vite dev server proxies `/api/*` to `http://localhost:8000` (see
`vite.config.js`), so once the Laravel backend is running, the frontend
will automatically switch from local fallback data to live API data — no
code changes needed.

To add the real resume PDF for the Resume Viewer / download button, drop it
at:

```
frontend/public/resume.pdf
```

### Build for production

```bash
npm run build     # outputs to frontend/dist
npm run preview   # serve the production build locally
```
